  <name>Task 1: Create GLB model validation utility (RETL-03)</name>
  <files>src/ar/utils/modelValidator.ts</files>
  <action>
Create `src/ar/utils/modelValidator.ts` that validates a GLB file before upload:

1. Define the result interface:
```typescript
export interface ModelValidationResult {
  valid: boolean;
  errors: string[];    // Blocking issues
  warnings: string[];  // Non-blocking advisories
}
```

2. Export an async function `validateGLBModel(file: File): Promise<ModelValidationResult>`:
   - Create a local URL via `URL.createObjectURL(file)`.
   - Load the model using GLTFLoader (same pattern as ModelLoader.ts).
   - Run these checks on the loaded scene:

   **Structure checks (errors):**
   - Model must contain at least one Mesh. If `gltf.scene` has zero meshes after traversal, error: "Model contains no 3D geometry (no meshes found)."
   - Model must have geometry with vertices. For each mesh, check `mesh.geometry.attributes.position` exists and has count > 0. Error: "Mesh '{name}' has no vertex data."

   **Normals check (warnings):**
   - For each mesh, check if `mesh.geometry.attributes.normal` exists. If missing, warning: "Mesh '{name}' has no normals -- lighting may appear incorrect."
   - Check for inverted normals by sampling: compute the bounding box center, then for a sample of up to 20 faces, check if the face normal points away from the center (dot product of normal with center-to-face vector should be > 0 for at least 60% of faces). If <60%, warning: "Mesh '{name}' may have inverted normals -- model could appear inside-out."

   **Scale/bounding box checks (errors and warnings):**
   - Compute bounding box via `new THREE.Box3().setFromObject(gltf.scene)`.
   - Get size via `box.getSize(new THREE.Vector3())`.
   - If any dimension is 0, error: "Model has zero size in one or more dimensions."
   - If max dimension > 100, error: "Model is extremely large (max dimension: {val} units). Expected models under 10 units. Re-export at a smaller scale."
   - If max dimension < 0.001, error: "Model is extremely small (max dimension: {val} units). It may not be visible in AR."
   - If max dimension > 10 but <= 100, warning: "Model is quite large ({val} units). This may cause scaling issues in AR."
   - If the aspect ratio (max dim / min dim) > 20, warning: "Model has extreme proportions ({ratio}:1). This may look distorted."

   **File size check (warnings):**
   - If `file.size > 10 * 1024 * 1024` (10MB), warning: "Model file is large ({sizeMB}MB). This may cause slow loading on mobile devices."
   - If `file.size > 25 * 1024 * 1024` (25MB), error: "Model file exceeds 25MB. This will cause poor performance on mobile."

   3. Clean up: after validation, traverse the loaded scene and dispose all geometry and materials. Revoke the object URL.
   4. Wrap the entire function in try-catch. If GLTFLoader throws, return `{ valid: false, errors: ["Failed to parse model file: {error.message}"], warnings: [] }`.

Use JSDoc to document the function and interface.
