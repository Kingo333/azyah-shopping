  <name>Task 1: Add model caching to ModelLoader (PERF-04)</name>
  <files>src/ar/core/ModelLoader.ts</files>
  <action>
Add a module-level `Map<string, ModelResult>` cache to ModelLoader.ts. When `loadModel(url)` is called:
1. Check if `modelCache.has(url)` -- if yes, clone the cached wrapper Group via `wrapper.clone(true)` (deep clone so each instance gets its own transform state) and return a new ModelResult with the clone and the original dims. This avoids re-downloading and re-parsing the GLB.
2. If not cached, load via GLTFLoader as current, store the result in the cache, then return a clone (not the original -- the original stays pristine in cache).
3. Export a `clearModelCache()` function that iterates the cache, disposes all geometry and materials in each cached wrapper, and clears the Map. This prevents memory leaks when the AR experience unmounts.

Important: The clone must deep-clone materials too. Use `wrapper.clone(true)` which clones the full hierarchy. Then traverse the clone and call `material.clone()` on each mesh so that per-instance opacity changes (via SceneManager.updateOpacity) do not bleed across cached copies.

Keep the existing ModelResult interface unchanged. The cache is transparent to callers -- same API signature.

Update the JSDoc to document the caching behavior.

<task type="auto">
  <name>Task 3: Add adaptive lighting from camera brightness (VIS-02)</name>
  <files>src/ar/core/SceneManager.ts, src/pages/ARExperience.tsx</files>
  <action>
Add adaptive lighting to SceneManager that adjusts ambient and directional light intensity based on camera feed brightness:

**In SceneManager.ts:**

1. Store references to the ambient and directional lights as private fields (`private ambientLight: THREE.AmbientLight` and `private dirLight: THREE.DirectionalLight`) instead of creating them inline in the constructor. This allows updating their intensity later.
2. Add a private `brightnessCanvas` (an OffscreenCanvas or a document.createElement('canvas') with small resolution 32x16) and its 2d context as private fields. These are created once in the constructor.
3. Add a public method `updateLightingFromVideo(video: HTMLVideoElement): void`:
   - Draw the video frame to the small canvas (32x16 resolution -- just enough to sample average brightness, not expensive).
   - Get the pixel data via `ctx.getImageData(0, 0, 32, 16).data`.
   - Compute average brightness: iterate pixels, compute luminance per pixel as `(0.299*r + 0.587*g + 0.114*b) / 255`, average across all pixels. This yields a 0-1 value.
   - Map brightness to light intensity: `ambientIntensity = 0.3 + brightness * 0.7` (range 0.3-1.0), `directionalIntensity = 0.4 + brightness * 0.8` (range 0.4-1.2). This ensures models are never completely dark but dim noticeably in low light.
   - Apply to lights: `this.ambientLight.intensity = ambientIntensity`, `this.dirLight.intensity = directionalIntensity`.
   - Set `this.dirty = true`.
   - Throttle internally: only sample every 500ms (store a `lastBrightnessSample` timestamp, skip if <500ms elapsed). Brightness changes slowly; sampling every frame wastes CPU.
4. Update `dispose()` to clean up the brightness canvas if needed.

**In ARExperience.tsx:**

5. In the render loop (Effect 1), after pose processing but before `sm.renderIfDirty()`, call `sm.updateLightingFromVideo(video)`. Since updateLightingFromVideo self-throttles to 500ms, calling it every frame is safe -- it will no-op most of the time.

Do NOT use OffscreenCanvas -- it is not supported in all mobile browsers. Use `document.createElement('canvas')` instead.
