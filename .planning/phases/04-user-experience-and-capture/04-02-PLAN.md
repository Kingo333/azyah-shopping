  <files>src/ar/core/SceneManager.ts, src/ar/capture/captureCompositor.ts, src/ar/capture/shareHandler.ts</files>
  <action>
1. In src/ar/core/SceneManager.ts, add `preserveDrawingBuffer: true` to the WebGLRenderer constructor options:
   ```typescript
   this.renderer = new THREE.WebGLRenderer({ canvas, alpha: true, preserveDrawingBuffer: true });
   ```
   This allows `canvas.toDataURL()` to capture the current Three.js frame. Without it, the canvas is cleared after each render and toDataURL returns a blank image. The performance cost is minimal (no measurable FPS impact on mobile for this use case since we already render-on-dirty at ~15fps).

2. Create src/ar/capture/captureCompositor.ts with one exported function:

   ```typescript
   export async function compositeCapture(
     video: HTMLVideoElement,
     overlayCanvas: HTMLCanvasElement,
   ): Promise<Blob>
   ```

   Implementation:
   - Create an offscreen canvas at the display resolution: `width = window.innerWidth`, `height = window.innerHeight`. Use `window.innerWidth * devicePixelRatio` and `window.innerHeight * devicePixelRatio` for higher quality on high-DPI screens, but cap DPR at 2 to avoid huge images.
   - Get 2D context from offscreen canvas.
   - Draw the video feed first. The video must be drawn mirrored (same as the CSS `scaleX(-1)` on the video element) and cropped to match the `object-fit: cover` behavior. To mirror: `ctx.translate(canvasWidth, 0); ctx.scale(-1, 1);` before drawImage, then restore.
   - For cover-crop: calculate the video source rect using the same logic as computeCoverCrop. The video's natural dimensions may differ from the display dimensions. Compute source x/y/w/h to crop the center of the video to fill the canvas.
   - Draw the Three.js overlay canvas on top: `ctx.drawImage(overlayCanvas, 0, 0, canvasWidth, canvasHeight)`. The overlay canvas already has alpha transparency so the video shows through.
   - Convert to Blob: `offscreen.toBlob(resolve, 'image/png')` wrapped in a Promise.
   - Return the Blob.

3. Create src/ar/capture/shareHandler.ts with one exported function:

   ```typescript
   export async function shareImage(blob: Blob): Promise<void>
   ```

   Implementation:
   - Convert Blob to File: `new File([blob], 'ar-tryon.png', { type: 'image/png' })`.
   - Try native share first using navigator.share (Web Share API Level 2 with files):
     ```typescript
     if (navigator.canShare && navigator.canShare({ files: [file] })) {
       await navigator.share({ files: [file], title: 'AR Try-On', text: 'Check out this look!' });
       return;
     }
     ```
   - If Web Share API not available or fails, fall back to download:
     ```typescript
     const url = URL.createObjectURL(blob);
     const a = document.createElement('a');
     a.href = url;
     a.download = 'ar-tryon.png';
     a.click();
     URL.revokeObjectURL(url);
     ```
   - Do NOT use Capacitor Share directly because it requires a file URI (not blob), which complicates the flow. The Web Share API works on both Capacitor WebView and regular browsers. If navigator.share is available in the Capacitor WebView (it is on iOS 15+ and Android Chrome), it will use the native share sheet automatically.
   - Wrap everything in try/catch. If the user cancels the share dialog (AbortError), silently ignore. For other errors, re-throw.
  </action>

2. Add state for capture flow:

   ```typescript
   const [capturedBlob, setCapturedBlob] = useState<Blob | null>(null);

   ```

3. Add capture handler function inside the component:
   ```typescript
   const handleCapture = async () => {
     if (!videoRef.current || !canvasRef.current || isCapturing) return;
     setIsCapturing(true);
     try {
       // Force a render so the overlay canvas has current frame
       sceneManagerRef.current?.renderIfDirty();
       const blob = await compositeCapture(videoRef.current, canvasRef.current);
       setCapturedBlob(blob);
     } catch (err) {
       console.error('Capture failed:', err);
     } finally {
       setIsCapturing(false);
     }
   };
   ```

4. Add share handler:
   ```typescript
   const handleShare = async () => {
     if (!capturedBlob) return;
     try {
       await shareImage(capturedBlob);
     } catch (err) {
       console.error('Share failed:', err);
     }
   };
   ```

5. Add capture button to the JSX. Place it in the right side of the screen, vertically centered, as a floating action button. Position it between the header and the product selector:

   ```tsx
   {/* Capture controls */}
   {trackingState === 'tracking_active' && (

   ```

6. Add a capture preview overlay. When capturedBlob is set, show a brief preview with share and dismiss buttons:

   ```tsx
   {capturedBlob && (
     <div className="absolute inset-0 z-30 bg-black/80 flex flex-col items-center justify-center gap-4 p-6">
       <img
         src={URL.createObjectURL(capturedBlob)}
         alt="AR capture preview"
         className="max-w-full max-h-[70vh] rounded-xl shadow-2xl object-contain"

         >
           Dismiss
         </Button>
       </div>
     </div>
   )}
   ```

   Note: The `URL.createObjectURL(capturedBlob)` creates an object URL each render. To avoid memory leaks, add a cleanup: use a `useMemo` or `useEffect` that revokes the URL when capturedBlob changes. Simplest approach: store the object URL in a ref and revoke on change:
   ```typescript
   const previewUrlRef = useRef<string | null>(null);
   // When capturedBlob changes, revoke old URL and create new one
   useEffect(() => {
     if (previewUrlRef.current) URL.revokeObjectURL(previewUrlRef.current);
     previewUrlRef.current = capturedBlob ? URL.createObjectURL(capturedBlob) : null;
     return () => { if (previewUrlRef.current) URL.revokeObjectURL(previewUrlRef.current); };
   }, [capturedBlob]);
   ```
   Then use `previewUrlRef.current` as the img src.
