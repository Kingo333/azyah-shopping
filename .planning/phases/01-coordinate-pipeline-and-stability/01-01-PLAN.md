Create `src/ar/utils/OneEuroFilter.ts` implementing the One Euro Filter algorithm (Casiez et al., 2012). This is a standalone pure-math class with no external dependencies.

The class must have:
- Constructor accepting `minCutoff` (default 1.0), `beta` (default 0.007), `dCutoff` (default 1.0)
- `filter(x: number, timestamp: number): number` method -- timestamp MUST be in seconds (not milliseconds). Add a JSDoc comment on the method specifying this.
- `reset(): void` method to clear state (used when switching products or re-initializing)
- Private `alpha(tE: number, cutoff: number): number` helper computing the smoothing factor
- Private state: `xPrev`, `dxPrev`, `tPrev`, `initialized` flag

Algorithm (from the 1 Euro Filter paper):
1. On first call, store value and return it unfiltered
2. On subsequent calls: compute time elapsed (tE), estimate derivative with low-pass, compute adaptive cutoff = minCutoff + beta * |derivative|, apply low-pass filter to value
3. If tE <= 0, return previous value (guard against duplicate timestamps)

Export the class as a named export. Add a JSDoc comment on the class explaining its purpose and linking to the paper.

Also export recommended parameter presets as constants:
```typescript
export const FILTER_PRESETS = {
  position: { minCutoff: 1.0, beta: 0.007, dCutoff: 1.0 },
  scale: { minCutoff: 0.5, beta: 0.001, dCutoff: 1.0 },
  rotation: { minCutoff: 0.8, beta: 0.004, dCutoff: 1.0 },
} as const;
```

Reference implementation is in 01-RESEARCH.md Pattern 2. Follow it closely.

Create `src/ar/utils/coordinateUtils.ts` with two pure functions and one interface. No external dependencies.

**Interface -- CoverCropInfo:**
```typescript
export interface CoverCropInfo {
  scaleX: number;   // how much of the video's X range is visible (0-1)
  scaleY: number;   // how much of the video's Y range is visible (0-1)
  offsetX: number;  // normalized offset cropped from each side in X
  offsetY: number;  // normalized offset cropped from each side in Y
}
```

**Function 1 -- computeCoverCrop:**
```typescript
export function computeCoverCrop(
  videoWidth: number, videoHeight: number,
  displayWidth: number, displayHeight: number
): CoverCropInfo
```
Implements object-fit:cover crop geometry:
- If video aspect > display aspect: video is wider, crop sides. scaleX = displayAspect / videoAspect, scaleY = 1, offsetX = (1 - scaleX) / 2, offsetY = 0
- If video aspect <= display aspect: video is taller, crop top/bottom. scaleX = 1, scaleY = videoAspect / displayAspect, offsetX = 0, offsetY = (1 - scaleY) / 2

Add guard: if any dimension is <= 0, return identity { scaleX: 1, scaleY: 1, offsetX: 0, offsetY: 0 }.

**Function 2 -- landmarkToWorld:**
```typescript
export function landmarkToWorld(
  landmark: { x: number; y: number },
  coverCrop: CoverCropInfo,
  visibleDims: { w: number; h: number },
  mirror: boolean = true
): { x: number; y: number }
```
Three-step transform:
1. Mirror X for selfie camera: `nx = mirror ? 1 - landmark.x : landmark.x` (handle ONCE here, nowhere else in the codebase)
2. Apply cover-crop: `nx = (nx - coverCrop.offsetX) / coverCrop.scaleX`, same for ny
3. Convert [0,1] to Three.js world units: `worldX = (nx - 0.5) * visibleDims.w`, `worldY = -(ny - 0.5) * visibleDims.h` (flip Y: screen Y is top-down, Three.js Y is bottom-up)

Add JSDoc comments explaining each function's purpose, parameters, and the coordinate transform pipeline.

Reference implementation is in 01-RESEARCH.md Pattern 1. Follow it closely.
